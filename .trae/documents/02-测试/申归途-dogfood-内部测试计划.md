# 申归途 Web 应用 Dogfood 内部测试计划

## 概要

对"申归途"抑郁症复发预防 React SPA 进行全面内部测试（dogfood），包括功能测试、UX 测试、边界测试，并对照产品设计方案进行完成度评估。

- **应用**：React 18 + Vite 5 + HashRouter + CSS Modules + LocalStorage（无后端）
- **页面**：8 个路由页面 + 4 个 BottomNav Tab + 浮动危机按钮
- **浏览器操作**：通过 MCP browser 工具（非 agent-browser CLI）

---

## 一、测试准备

### 1.1 启动服务器

```bash
cd /sessions/69f1935cddc02792ee45ea27/workspace/shen-gui-tu
npx vite preview --port 4173 --host 0.0.0.0
```

### 1.3 HashRouter 路由格式

所有路由为 `http://localhost:4173/#/pathname`，导航时必须带 `#` 前缀。

## 三、产品设计完成度评估

对照设计方案 6 大核心模块逐一评估：

| 模块 | 完成度 | 关键缺失 |
|------|--------|---------|
| 模块一：复发预警与动态监测 | ~70% | 被动采集（需穿戴设备）、个人基线校准 |
| 模块二：药物管理与依从性支持 | ~75% | 服药日历、提醒系统、应对策略推送 |
| 模块三：数字心理干预与技能训练 | ~50% | 课程数量不足（3/12+）、缺 MBCT 正念课、无音视频 |
| 模块四：社会支持与生活重建 | ~15% | 社交功能需后端（仅静态资源信息） |
| 模块五：家庭/照护者支持 | 0% | 需后端+多用户系统 |
| 模块六：个性化复发预防计划 | ~60% | 引导式创建、定期回顾机制 |
| **总体** | **~45%** | 前端可实现部分约 60-65% |

## 四、已知代码问题（源码审查发现，测试时重点验证）

1. **Medication.jsx**：服药按钮文案无论状态都显示"已服用"，应区分"服用"和"已服用"
2. **Dashboard.jsx**：streak 计算为近 7 天记录数，非真正连续天数
3. **Crisis.jsx**：呼吸练习闭包可能引用旧值，导致只执行一次
4. **SideEffectTracker** 与 **useMedMoodCorrelation** 的周次计算函数重复但实现不同，可能导致不一致

## 六、注意事项

1. **数据隔离**：每轮测试前 `localStorage.clear()`
2. **动画等待**：RiskGauge/依从率动画 ~600ms，Toast 2500ms，呼吸练习 19s/周期
3. **控制台监控**：每页测试后检查 `browser_console_messages`
4. **截图规范**：完整应用区域（顶到底部导航），交互操作前后各一张
5. **数据注入**：测试联动功能时需注入历史签到+副作用数据到 localStorage
6. **移动端模拟**：应用 CSS 限制 480px 宽度，桌面自动居中
