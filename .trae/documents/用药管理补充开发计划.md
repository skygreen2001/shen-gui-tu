# 用药管理模块补充开发计划

## 概述

基于需求文档（设计方案 4.2节），当前用药管理模块仅实现了"每日服药记录 + 依从率"核心功能。需补充副作用追踪、药物知识库、药物-情绪联动分析、减药导航四大功能。

## 当前状态

| 功能 | 状态 | 说明 |
|------|------|------|
| 每日服药打卡 | ✅ 已实现 | 已服/跳过 + 正向反馈 |
| 月度依从率 | ✅ 已实现 | 环形图 + 颜色分级 |
| 添加/删除药物 | ✅ 已实现 | 表单输入 |
| 副作用追踪 | ❌ 缺失 | 每周结构化自评 |
| 药物知识库 | ❌ 缺失 | 通俗化药物说明 |
| 药物-情绪联动 | ❌ 缺失 | 时序关联分析 |
| 减药导航 | ❌ 缺失 | 渐进减药计划 |

## 开发优先级与依赖

```
P0 ── 药物知识库（纯数据+展示，最快交付）
P0 ── 副作用追踪（建立数据采集能力）
P1 ── 药物-情绪联动分析（依赖副作用数据）
P2 ── 减药导航（最复杂，依赖前三个功能）
```

## 实施步骤

### 步骤 1：药物知识库（P0）

**新增文件：**
- `src/data/medicationKnowledge.js` — 药物知识静态数据（草酸艾司西酞普兰、喹硫平等常见抗抑郁药物的通俗化说明：作用机制、副作用、起效时间、注意事项、FAQ）
- `src/pages/Medication/MedKnowledge.jsx` — 手风琴式知识展示组件
- `src/pages/Medication/MedKnowledge.module.css`

**修改文件：**
- `src/pages/Medication/Medication.jsx` — 每个药物卡片增加「了解此药」按钮，点击展开知识面板（useState 视图切换，非新路由）
- `src/pages/Medication/Medication.module.css` — 新增入口按钮样式

### 步骤 2：副作用追踪（P0）

**新增文件：**
- `src/hooks/useSideEffects.js` — 副作用数据 CRUD hook（LocalStorage key: `sgt-side-effects`），包含 submitWeeklyReport、getWeeklyHistory、getSideEffectTrend
- `src/pages/Medication/SideEffectTracker.jsx` — 7维度评分组件（体重变化/嗜睡/性功能/胃肠反应/失眠/食欲变化/焦虑躁动），0-4 五级量表 + 备注
- `src/pages/Medication/SideEffectTracker.module.css`

**修改文件：**
- `src/pages/Medication/Medication.jsx` — 页面底部增加「副作用周报」入口卡片
- `src/pages/Medication/Medication.module.css`

### 步骤 3：药物-情绪联动分析（P1）

**新增文件：**
- `src/hooks/useMedMoodCorrelation.js` — 跨数据源关联 hook，读取 `sgt-checkins`（情绪）+ `sgt-side-effects`（副作用）+ `sgt-medications`（依从率），生成按周汇总数据
- `src/pages/Medication/MoodCorrelation.jsx` — 双折线图展示 + 文字洞察（纯 CSS/SVG，不引入图表库）
- `src/pages/Medication/MoodCorrelation.module.css`

**修改文件：**
- `src/pages/Medication/Medication.jsx` — 增加「情绪联动」入口卡片

### 步骤 4：减药导航（P2）

**新增文件：**
- `src/hooks/useTaperPlan.js` — 减药计划 CRUD hook（LocalStorage key: `sgt-taper-plans`），包含 createPlan、updatePhaseStatus、generateDoctorReport
- `src/pages/Medication/TaperNavigator.jsx` — 减药计划创建 + 垂直时间线进度展示
- `src/pages/Medication/TaperNavigator.module.css`
- `src/pages/Medication/TaperReport.jsx` — 医生报告生成组件（纯文本，可复制发送给医生）
- `src/pages/Medication/TaperReport.module.css`

**修改文件：**
- `src/pages/Medication/Medication.jsx` — 增加「减药计划」入口卡片（有药物时显示）

## 架构决策

1. **不新增路由** — 四个功能作为 Medication 页面的子视图（useState 切换），保持 BottomNav「疗愈」Tab 不变
2. **不引入图表库** — 联动分析使用纯 CSS + SVG 自制轻量图表，避免增加包体积
3. **数据隔离** — 三个新数据源使用独立 LocalStorage key（`sgt-side-effects`、`sgt-taper-plans`），关联分析在运行时跨 key 读取

## 验证步骤

1. `npx vite build` 构建无错误
2. 药物知识库：点击「了解此药」→ 展开手风琴面板 → 显示药物说明
3. 副作用追踪：点击入口 → 7维度评分 → 提交 → Toast 反馈 → 历史记录可查看
4. 联动分析：数据不足时显示「数据积累中」提示；有数据时显示双折线图 + 洞察文字
5. 减药导航：创建计划 → 时间线展示 → 生成医生报告 → 可复制文本
