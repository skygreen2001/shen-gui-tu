# 心理干预课程音视频功能实施方案

## 概要

为 CBT/MBCT 课程增加"引导式练习"能力。纯前端无法生成真实音频/视频文件，但可以通过 **Web Speech API 语音合成（TTS）** + **CSS 动画引导界面** 实现等效的"引导音频"体验。用户点击播放后，系统朗读引导词并配合呼吸圆圈/身体扫描动画。

**核心思路**：不依赖外部音频文件，用浏览器内置 TTS + 定时器 + CSS 动画实现"伪音频"引导。

---

## 一、技术方案

### 方案选择：Web Speech API TTS + 引导动画

| 方案 | 优点 | 缺点 |
|------|------|------|
| ❌ 外部音频文件 | 音质好 | 需录制/托管，工作量大 |
| ❌ 内嵌 base64 音频 | 离线可用 | 体积大（几分钟音频几MB） |
| ✅ **Web Speech API TTS** | 零资源依赖，代码量小 | 音质依赖系统TTS引擎，需在线 |
| ✅ **CSS 动画引导** | 复用 Crisis 呼吸动画模式 | — |

### 适用课程步骤

不需要所有步骤都加音视频，仅对以下类型有意义：

| 课程 | 适用步骤 | 引导类型 |
|------|---------|---------|
| 身体扫描入门 | "5分钟身体扫描"练习 | 身体部位逐段扫描动画 + TTS |
| 三分钟呼吸空间 | "跟随引导练习" | 复用 Crisis 呼吸圆圈 + TTS |
| 与低落情绪共处 | "情绪观察练习" | 呼吸圆圈 + TTS |
| 正念自我关怀 | "自我关怀冥想" | 呼吸圆圈 + TTS |

共 **4 个步骤** 需要引导式练习功能。

## 二、数据结构扩展

### 步骤新增 `guided` 字段

在 `cbtContent.js` 中，为需要引导的练习步骤添加 `guided` 对象：

```js
{
  type: '练习',
  title: '5分钟身体扫描',
  guided: {
    type: 'bodyScan',  // 'breathing' | 'bodyScan' | 'meditation'
    segments: [
      { text: '注意呼吸3次', duration: 5000 },
      { text: '将注意力移到头顶，感受那里的感觉', duration: 8000 },
      { text: '缓慢移动到额头、眼睛、脸颊', duration: 8000 },
      // ...
    ]
  },
  body: '...'  // 保留原有文字版
}
```

### 引导类型定义

| type | 动画 | 适用 |
|------|------|------|
| `breathing` | 呼吸圆圈（复用 Crisis 模式） | 呼吸空间、情绪共处、自我关怀 |
| `bodyScan` | 身体部位高亮滚动 | 身体扫描入门 |
| `meditation` | 静态圆圈 + 柔和脉冲 | 通用冥想 |

## 三、实施步骤

### Step 1：扩展 cbtContent.js 数据

**文件**：`src/data/cbtContent.js`

为 4 个练习步骤添加 `guided` 字段：
- 课程4（身体扫描）→ 步骤2 "5分钟身体扫描"：type=`bodyScan`，8个身体部位段落
- 课程5（呼吸空间）→ 步骤2 "跟随引导练习"：type=`breathing`，3个阶段
- 课程6（情绪共处）→ 步骤2 "情绪观察练习"：type=`meditation`，5个观察段落
- 课程9（自我关怀）→ 步骤2 "自我关怀冥想"：type=`meditation`，6个引导段落

### Step 2：创建 GuidedPlayer 组件

**新建文件**：`src/components/GuidedPlayer/GuidedPlayer.jsx` + `.module.css`

功能：
- 接收 `guided` 对象作为 props
- 播放/暂停按钮
- 当前段落文字显示
- 进度指示（第几段/共几段）
- 倒计时显示
- TTS 朗读当前段落文字
- CSS 动画（呼吸圆圈 or 身体扫描滚动）
- 播放完成后显示"练习结束"

TTS 实现：
```js
const utterance = new SpeechSynthesisUtterance(text)
utterance.lang = 'zh-CN'
utterance.rate = 0.8  // 慢速，适合冥想
speechSynthesis.speak(utterance)
```

### Step 3：修改 CBTCourse.jsx 集成 GuidedPlayer

**文件**：`src/pages/CBTCourse/CBTCourse.jsx`

在步骤内容渲染中，检测当前步骤是否有 `guided` 字段：
- 有 `guided` → 显示 GuidedPlayer 组件（替代纯文字）
- 无 `guided` → 保持原有文字渲染

```jsx
{step.guided ? (
  <GuidedPlayer guided={step.guided} />
) : (
  <div className={styles.stepBody}>...</div>
)}
```

### Step 4：构建验证

`npx vite build` 确认 0 错误。

## 五、边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 浏览器不支持 Speech API | 显示"您的浏览器不支持语音功能"，仍可看文字+动画 |
| 用户跳过引导 | 提供"跳过引导"按钮，回到文字模式 |
| TTS 朗读时长与 duration 不一致 | 以 duration 定时器为准，TTS 作为辅助 |
| 用户中途退出课程 | cleanup 中调用 `speechSynthesis.cancel()` |
