# 心理干预模块（合并文档）

> 本文档合并了心理干预模块的差距分析补充计划和音视频功能实施方案。

## 一、差距分析

### 概要

设计方案要求 12+ 门三层递进课程（MBCT + CBT），当前仅实现 3 门纯 CBT 课程。本计划分析差距并规划可落地的补充方案。

### 课程体系对比

| 层级 | 设计要求 | 当前实现 | 差距 |
|------|---------|---------|------|
| **第一层：基础技能** | 4门（认识情绪、身体扫描、自动思维识别、呼吸空间） | 2门（思维模式识别≈自动思维识别、认知重构≈认识情绪） | 缺身体扫描、呼吸空间 |
| **第二层：深化技能** | 5门（与低落共处、行为激活、认知灵活性、人际效能、正念自我关怀） | 1门（行为激活，内容较简略） | 缺4门 |
| **第三层：长期维护** | 4门（复发预防计划、预警信号手册、危机工具箱、意义重建） | 0门（WRAP页面独立存在但非课程形式） | 缺4门 |
| **合计** | **13门** | **3门** | **缺10门** |

### 框架能力对比

| 能力 | 设计要求 | 当前状态 |
|------|---------|---------|
| 渐进解锁 | ✅ | ✅ 已实现（prerequisites） |
| 微学习步骤制 | ✅ | ✅ 已实现（内容/练习/反思） |
| 练习重于理论 | ✅ | ✅ 已实现 |
| 灵活进度 | ✅ | ✅ 已实现（LocalStorage） |
| 多模态（音频/视频） | 文字+音频+视频 | ❌ 仅纯文本 |
| 文化适配（上海场景） | 中国/上海本地化 | ❌ 通用场景 |

### 差距优先级排序

| 优先级 | 缺失内容 | 可行性 | 理由 |
|--------|---------|--------|------|
| **P0** | 补充 MBCT 正念课程（身体扫描、呼吸空间、与低落共处、正念自我关怀） | ✅ 纯文本即可 | 设计方案核心亮点，MBCT可降低5年复发率42% |
| **P0** | 补充第二层深化课程（认知灵活性、人际效能） | ✅ 纯文本即可 | CBT体系完整性 |
| **P1** | 呼吸空间引导音频（复用Crisis页4-7-8呼吸动画思路） | ✅ 可用定时器+文字引导 | 多模态入门，难度低 |
| **P1** | 第三层课程（复发预防计划、预警手册、危机工具箱） | ⚠️ 与WRAP页面功能重叠 | 可整合为"WRAP进阶课程"引导用户完善WRAP |
| **P2** | 意义重建与价值探索（ACT模块） | ✅ 纯文本即可 | 第三层最独特的课程 |
| **P2** | 文化适配（上海本地化案例） | ✅ 修改现有课程文本 | 提升代入感 |

**文件**：`src/data/cbtContent.js`

```js
{ id: 1, title: '...', tier: 1, tag: 'CBT', ... }
{ id: 4, title: '身体扫描入门', tier: 1, tag: 'MBCT', ... }
```

## 三、引导式练习（音视频替代方案）

### 技术方案

纯前端无法生成真实音频/视频文件，通过 **Web Speech API 语音合成（TTS）** + **CSS 动画引导界面** 实现等效的"引导音频"体验。

| 方案 | 优点 | 缺点 |
|------|------|------|
| ❌ 外部音频文件 | 音质好 | 需录制/托管，工作量大 |
| ❌ 内嵌 base64 音频 | 离线可用 | 体积大（几分钟音频几MB） |
| ✅ **Web Speech API TTS** | 零资源依赖，代码量小 | 音质依赖系统TTS引擎 |
| ✅ **CSS 动画引导** | 复用 Crisis 呼吸动画模式 | — |

### 适用课程步骤

| 课程 | 适用步骤 | 引导类型 |
|------|---------|---------|
| 身体扫描入门 | "5分钟身体扫描"练习 | 身体部位逐段扫描动画 + TTS |
| 三分钟呼吸空间 | "跟随引导练习" | 复用 Crisis 呼吸圆圈 + TTS |
| 与低落情绪共处 | "情绪观察练习" | 呼吸圆圈 + TTS |
| 正念自我关怀 | "自我关怀冥想" | 呼吸圆圈 + TTS |

共 **4 个步骤** 需要引导式练习功能。

### 数据结构扩展

在 `cbtContent.js` 中，为需要引导的练习步骤添加 `guided` 对象：

```js
{
  type: '练习',
  title: '5分钟身体扫描',
  guided: {
    type: 'bodyScan',  // 'breathing' | 'bodyScan' | 'meditation'
    segments: [
      { text: '注意呼吸3次', duration: 5000 },
      { text: '将注意力移到头顶，感受那里的感觉', duration: 8000 },
      // ...
    ]
  },
  body: '...'  // 保留原有文字版
}
```

### 引导类型定义

| type | 动画 | 适用 |
|------|------|------|
| `breathing` | 呼吸圆圈（复用 Crisis 模式） | 呼吸空间、情绪共处、自我关怀 |
| `bodyScan` | 身体部位高亮滚动 | 身体扫描入门 |
| `meditation` | 静态圆圈 + 柔和脉冲 | 通用冥想 |

### GuidedPlayer 组件

**新建文件**：`src/components/GuidedPlayer/GuidedPlayer.jsx` + `.module.css`

功能：
- 接收 `guided` 对象作为 props
- 播放/暂停按钮
- 当前段落文字显示
- 进度指示（第几段/共几段）
- 倒计时显示
- TTS 朗读当前段落文字
- CSS 动画（呼吸圆圈 or 身体扫描滚动）
- 播放完成后显示"练习结束"

### 边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 浏览器不支持 Speech API | 显示"您的浏览器不支持语音功能"，仍可看文字+动画 |
| 用户跳过引导 | 提供"跳过引导"按钮，回到文字模式 |
| TTS 朗读时长与 duration 不一致 | 以 duration 定时器为准，TTS 作为辅助 |
| 用户中途退出课程 | cleanup 中调用 `speechSynthesis.cancel()` |

## 四、不做的事项

| 项目 | 原因 |
|------|------|
| 音频/视频内容 | 需要实际录制或外部资源，纯前端无法生成高质量音频 |
| 新建独立页面 | 复用现有 CBTCourse 组件即可，不增加路由复杂度 |
| ACT 深度课程 | 仅保留1门入门级，ACT 需专业引导不适合纯自助 |
| 文化适配全面重写 | 工作量大，新课程直接使用上海场景即可，旧课程暂不改 |
